## 🔥 چرا CF Scanner؟

بیشتر اسکنرهایی که توی گروه‌ها و کانال‌ها دست به دست می‌شن، فقط یه کار بلدن: **پینگ گرفتن از یه مشت IP خام.** همین. بعدش خودت باید بنشینی دستی، IP به IP، توی کانفیگت جاش کنی، تست کنی، ببینی کار می‌کنه یا نه — کاری که برای پیدا کردن ۲۰ تا IP سالم می‌تونه نیم ساعت وقتت رو بگیره.

CF Scanner یه ابزار دیگه نیست؛ یه **خط تولید کامل کانفیگ** است:

> پینگ بگیر → IP های تمیز رو پیدا کن → خودش برات، روی **همون کانفیگ خودت**، با همون UUID/پسورد/SNI/path، یه کانفیگ جدا برای هر IP بسازه → کپی کن، بچسبون توی V2RayNG/NekoBox، تمام.

دیگه لازم نیست خودت دستی IP رو جا‌به‌جا کنی. دیگه لازم نیست به یک دامنه یا یک سرویس‌دهنده خاص محدود باشی. و مهم‌تر از همه — وقتی اسکنر می‌گه «این IP زنده‌ست»، واقعاً منظورش این نیست که فقط پورت بازه؛ تا جایی پیش می‌ره که از **داخل خود Xray** یک اتصال واقعی برقرار می‌کند و جواب HTTP می‌گیرد.

## ✅ همه‌ی قابلیت‌ها

### 🗂 تب «کانفیگ»
- پارس کردن مستقیم لینک‌های اشتراک‌گذاری: **VLESS، VMess، Trojan**
- پشتیبانی کامل از انواع شبکه: `ws`، `grpc`، `h2`، `httpupgrade`، `splithttp`، `tcp`
- پشتیبانی از TLS و **Reality** (fingerprint، shortId، publicKey، spiderX)
- تشخیص خودکار SNI، path، host header، ALPN، allowInsecure
- تبدیل خودکار لینک به JSON استاندارد Xray، قابل مشاهده و ویرایش
- ذخیره/بارگذاری کانفیگ به‌صورت فایل JSON
- کار با **هر دامنه‌ای** — نه فقط دامنه‌های پشت کلادفلر

### 📡 تب «تست Ping»
- وارد کردن هر رنج IP به فرمت CIDR (نه فقط رنج‌های کلادفلر — رنج هر سرویس‌دهنده‌ای که بخوای)
- دکمه‌های آماده برای رنج‌های پرکاربرد کلادفلر
- اسکن چند‌نخی (Multi-thread) با تعداد Thread و Timeout قابل تنظیم
- محدودسازی تعداد IP اسکن‌شده (انتخاب تصادفی از رنج برای سرعت بیشتر)
- نمایش زنده‌ی نتایج با رنگ‌بندی بر اساس سرعت (سبز/زرد/قرمز)
- مرتب‌سازی خودکار بر اساس کمترین Ping
- **ذخیره‌ی IP های تمیز** در یک فایل تمیز و ساده (یک IP در هر خط)
- **ساخت خودکار کانفیگ برای تمام IP های تمیز** بر اساس کانفیگی که در تب «کانفیگ» وارد کردی — مثلاً اگر ۲۰ IP سالم پیدا شد، ۲۰ کانفیگ جدا با همان IP ها ساخته می‌شود، بدون اینکه هیچ پارامتر دیگری (UUID، پسورد، SNI، path) تغییر کند
- **کپی همه‌ی کانفیگ‌های ساخته‌شده در یک کلیک** به کلیپ‌بورد
- **ذخیره‌ی همه‌ی کانفیگ‌ها در یک فایل txt** برای استفاده بعدی یا اشتراک‌گذاری
- **دوبار کلیک روی هر IP** در جدول نتایج → کانفیگ همان IP به‌تنهایی فوراً در کلیپ‌بورد کپی می‌شود

### 🔒 تب «تست V2Ray / Xray»
- اجرای واقعی Xray در پس‌زمینه برای هر IP (نه فقط ping ساده!)
- ساخت خودکار inbound SOCKS5 محلی + routing صحیح
- تست واقعی اتصال HTTP از طریق پروکسی، بدون نیاز به curl یا ابزار جانبی
- مدیریت خودکار پورت‌های آزاد (بدون تداخل، حتی در تست پشت‌سرهم)
- **فیلد «دامنه تست» اختصاصی** — اگر دامنه واقعی با SNI فرق داشت یا دامنه‌ات پشت کلادفلر نبود، خودت می‌تونی override کنی
- لاگ رنگی و زمان‌دار از نتیجه‌ی هر تست
- وارد کردن IP مستقیم از نتایج تب Ping یا به‌صورت دستی
- ذخیره‌ی کامل لاگ تست در فایل
- مخفی‌سازی پنجره‌ی Xray در ویندوز (بدون پاپ‌آپ آزاردهنده‌ی cmd)

### 🎨 ظاهر و تجربه‌ی کاربری
- تم اختصاصی **سایبرپانک نئونی** — فیروزه‌ای، سبز نئون، صورتی/بنفش نئون روی پس‌زمینه‌ی مشکی
- رابط کاربری تب‌بندی‌شده و تمیز، مناسب کاربر تازه‌کار و حرفه‌ای
- نوار وضعیت زنده با درصد پیشرفت اسکن

## 📋 راهنمای سریع استفاده

1. در تب «کانفیگ»، لینک vless/vmess/trojan خودت رو paste کن و «تجزیه‌ی کانفیگ» رو بزن
2. به تب «تست Ping» برو، رنج CIDR (مال خودت یا یکی از رنج‌های آماده‌ی کلادفلر) رو انتخاب کن و «شروع اسکن» بزن
3. وقتی اسکن تموم شد، یا «📋 کپی کانفیگ برای IP های تمیز» رو بزن (همه‌ی کانفیگ‌ها با هم کپی می‌شن)، یا روی هر IP که خواستی دوبار کلیک کن (فقط همون یکی کپی می‌شه)
4. کانفیگ‌های کپی‌شده رو مستقیم توی V2RayNG / NekoBox / هر کلاینت دیگه پیست کن

## ⚠️ نکات
- برای تست V2Ray نیازی به curl نیست (پیاده‌سازی با socket خام)
- `xray.exe` باید در همان پوشه یا با مسیر کامل مشخص شود
- پورت محلی پیش‌فرض: `10808`
- ساخت دسته‌جمعی کانفیگ فقط برای لینک‌های paste‌شده (vless/vmess/trojan) کار می‌کند، نه برای فایل JSON بارگذاری‌شده
